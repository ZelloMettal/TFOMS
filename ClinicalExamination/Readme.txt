Проект выполнен на WinForms на платформе .NetFrameWork 4.8.1
На базе MSSQL Server взаимодействие с БД происходит с помощью технологии ADO.NET

UI представлет из себя главную форму(MainForm) с кнопкой "RUN" для запуска выполнения процедур
и RichTextBox для отображения результатов выполнения процедур

Содержание проекта:
В папке "INPUT" хранится тестовый XML-файл(XML_Test.xml) с данными для добавления в БД
и XSD-схема(XDS_TEST.xsd) для валидации.
В папке "SQL" содержатся файлы:
 - DB Schema - Полная схема базы данных
 - XSD Schema - запрос на сосздание XSD-схема в XML SCHEMA COLLECTION
 - SQL_Validator - запрос на создание хранимой процедуры валидации XML-файла и XSD-схемы
 - SQL_Check_CountPERS - запрос на создание хранимой процедуры проверки соответствия полей KOL_M и KOL_W c PERS
 - SQL_Isert_Data - запрос на создание хранимой процедуры для добавления данных из XML-файла в БД

В папке "Classes" содержатся классы для выполнения процедур на верхнем уровне:
 - DBWorker - класс для работы с БД
 - XMLEntity - набор сущностей для десериализации XML-файла
 - XMLDeserialize - класс десериализатор
 - ValidatorXML - класс-валидатор XML-файла и XSD-схемы

Принцип работы:

По нажатию на кнопку "RUN" последовательно выполняется операций на уровне бизнес логике, а затем на уровне БД
1. Проверка валидации XML-файла и XSD-схемы на уровне ПО
2. Проверка соответствия полей KOL_M, KOL_W c PERS на уровне ПО
Последующие операции выполняются по средством вызова хранимых процедур в БД
3. Проверка валидации XML-файла и XSD-схемы на уровне БД
4. Проверка соответствия полей KOL_M, KOL_W c PERS на уровне БД
5. Добавление данных в БД
На каждом этапе в RichTextBox выводятся результаты проверки
Если на каком-то из этапов проверка не прошла успешно, то вся процедура отменяется с отображением причины отмены

P.S. Для упрощения реализации все пути к файлам указаны статично из папки "INPUT" в корне приложения, 
а также стора подключения к базе хранится в файле самого приложения.
В базе данных в таблицу PERS было добавлено поле EVENT_ID для создания коректной связи таблиц EVENT и PERS,
а также обеспечения нормализации данных.
